**2. 核心超参数 (针对 40GB 显存极限压缩)** 
由于显存非常受限，配置大量使用了**梯度累加 (Gradient Accumulation)** 与 **Gradient Checkpointing** 来防范 OOM。

| 参数名称 | 第一阶段 (SFT) | 第二阶段 (RM) | 第三阶段 (PPO/RLOO) |
| :--- | :--- | :--- | :--- |
| **微调策略 (PEFT)** | LoRA (r=8) | LoRA (r=8) | LoRA (r=8) |
| **加载精度 (参数: dtype)** | 隐式 bfloat16 (读基座 config) | 隐式 bfloat16 (读基座 config) | bfloat16 (手动指明) |
| **训练精度 (参数: bf16)** | bfloat16 (纯正混合精度) | bfloat16 (纯正混合精度) | bfloat16 (刚刚紧急补救) |
| **Gradient Checkpointing**| True | True | True |
| **Batch Size (每张卡)** | 2 | 2 | 1 |
| **梯度累加步数** | 8 | 8 | 8 |
| **等效总 Batch Size** | **16** (2×8) | **16** (2×8) | **8** (1×8) |
| **Epoch (总训练轮数)** | 3 | 1 | - |
| **Max Steps (强行截断步数)**| - | - | 50 |

> 💡 **参数解析附注**：
> RM (奖励模型) 阶段的序列长度 `2048 / 512` 分别指代 `--max_source_length` 和 `--max_target_length`。
> 与 SFT 阶段不同，RM 的训练数据是“一问两答”结构。前置 2048 Token 专门用于装填患者漫长复杂的“病历提问”，而后置 512 Token 用于截断参与打分的“医生回答”。两者拼接后，单条数据的极端极限长度可达 2560 Tokens，此精细拆分能在最大程度保留病历细节的同时，严格控制 40GB 显卡不发生 OOM。
